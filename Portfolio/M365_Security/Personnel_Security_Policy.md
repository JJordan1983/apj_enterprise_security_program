# Personnel Security Policy  
APJ SecureCloud  
Version 1.0

## 1. Purpose
Ensures APJ personnel with access to FedRAMP Low data and systems are
screened, trained, and monitored per NIST and FedRAMP expectations.

## 2. Screening
All personnel must complete:
- Employment verification
- Background screening
- NDA agreement

## 3. Access Restrictions
Personnel may not access:
- Azure Gov administrative resources without approval
- Customer metadata without authorization

## 4. Training
Must complete:
- Annual security awareness training
- Zero Trust training
- IR tabletop participation (as required)
- Role-based training for admins

## 5. Termination
Access removed within **24 hours** of separation.

## 6. Approval
Approved by APJ Security Program Lead.
