# System and Services Acquisition Policy  
APJ SecureCloud  
Version 1.0

## 1. Purpose
Ensures all acquired software, services, and components meet FedRAMP Low
security requirements.

## 2. Requirements
APJ must:
- Use FedRAMP-authorized services
- Evaluate vendor security posture
- Require MFA and identity controls
- Prevent acquisition of unsupported software

## 3. Development Requirements
- CI/CD security scanning
- Code peer review
- SBOM optional
- Secure coding practices

## 4. Approval
Approved by APJ Security Program Lead.
